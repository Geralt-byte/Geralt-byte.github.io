## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值 `target` 的那两个整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

## 示例

### 示例 1：
```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
```

### 示例 2：
```
输入：nums = [3,2,4], target = 6
输出：[1,2]
```

### 示例 3：
```
输入：nums = [3,3], target = 6
输出：[0,1]
```

## 解题思路

### 方法一：暴力枚举

最直观的方法是使用双重循环，枚举数组中的所有两数组合，检查它们的和是否等于目标值。

**算法步骤：**
1. 外层循环从第一个元素开始遍历到倒数第二个元素
2. 内层循环从外层循环的下一个元素开始遍历到最后一个元素
3. 检查两个元素的和是否等于目标值
4. 如果等于，返回两个元素的索引

**复杂度分析：**
- 时间复杂度：O(n²)，其中 n 是数组的长度。嵌套循环最多执行 n(n-1)/2 次。
- 空间复杂度：O(1)，只需要常数级别的额外空间。

### 方法二：哈希表

使用哈希表来存储已经遍历过的元素及其索引，可以在O(1)的时间内查找元素。

**算法步骤：**
1. 创建一个空的哈希表
2. 遍历数组中的每个元素
3. 对于每个元素，计算 `target - nums[i]`
4. 检查这个差值是否存在于哈希表中
5. 如果存在，返回哈希表中的索引和当前索引
6. 如果不存在，将当前元素及其索引存入哈希表

**复杂度分析：**
- 时间复杂度：O(n)，其中 n 是数组的长度。我们只需要遍历一次数组。
- 空间复杂度：O(n)，哈希表最多需要存储 n-1 个元素。

## 代码实现

### 方法一：暴力枚举

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        int n = nums.length;
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                if (nums[i] + nums[j] == target) {
                    return new int[]{i, j};
                }
            }
        }
        return new int[0];
    }
}
```

### 方法二：哈希表

```java
class Solution {
    public int[] twoSum(int[] target, int[] nums) {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            if (map.containsKey(complement)) {
                return new int[]{map.get(complement), i};
            }
            map.put(nums[i], i);
        }
        return new int[0];
    }
}
```

## 算法比较

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|------------|------------|----------|
| 暴力枚举 | O(n²) | O(1) | 数据规模小 |
| 哈希表 | O(n) | O(n) | 数据规模大 |

## 总结

两数之和是哈希表应用的经典题目，通过哈希表可以将时间复杂度从O(n²)优化到O(n)。在实际应用中，哈希表方法通常是更优的选择，特别是当数组规模较大时。

## 扩展思考

1. 如果数组是有序的，可以使用双指针法，时间复杂度也是O(n)，但空间复杂度可以降到O(1)。
2. 如果题目要求返回的是元素的值而不是索引，方法会更加多样。
3. 在实际面试中，除了代码实现，还应该讨论边界情况的处理。

## 边界情况

- 数组长度为2的情况
- 目标值为负数的情况
- 数组中有重复元素的情况
- 多个解的情况（虽然题目保证唯一解）

## 性能优化

对于哈希表方法，可以进一步优化：
- 提前判断数组长度是否小于2
- 处理整数溢出的情况
- 使用更高效的哈希表实现
