## 题目描述

给你一个整数数组 `nums` ，找到其中最长严格递增子序列的长度。

**子序列** 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，`[3,6,2,7]` 是数组 `[0,3,1,6,2,2,7]` 的子序列。

## 解题思路

### 方法一：动态规划 (O(n²))

这是最经典的做法，也是最容易理解的。

**1. 状态定义：**
定义 `dp[i]` 为：**以 `nums[i]` 结尾的最长上升子序列的长度**。

**2. 状态转移方程：**
遍历 `i` 之前的所有元素 `j` (`0 <= j < i`)：
- 如果 `nums[i] > nums[j]`，说明 `nums[i]` 可以接在 `nums[j]` 后面形成一个新的递增序列。
- 此时状态转移方程为：`dp[i] = max(dp[i], dp[j] + 1)`。

**3. 初始化：**
每一个元素至少可以单独成为一个子序列，所以 `dp[i]` 初始值都为 `1`。

**4. 结果：**
遍历 `dp` 数组，取最大值。

#### Python 代码实现

n = len(nums)
dp = [1] * n

for i in range(n):
    for j in range(i):
        if nums[i] > nums[j]:
            dp[i] = max(dp[i], dp[j] + 1)

return max(dp)

**复杂度分析：**
- 时间复杂度：$O(n^2)$
- 空间复杂度：$O(n)$

### 方法二：贪心 + 二分查找 (O(n log n))

这个方法比较难想到，但效率更高。核心思想是维护一个“最小的结尾数组”。

**思想：**
我们并不关心具体的序列是什么，我们只关心长度。我们维护一个数组 `tails`，`tails[i]` 表示长度为 `i+1` 的递增子序列中，**结尾元素的最小值**。

**过程：**
遍历 `nums` 中的每个数字 `x`：
1. 如果 `x` 比 `tails` 中的所有数都大，说明可以接在后面形成更长的序列，`tails.append(x)`。
2. 否则，我们找到 `tails` 中第一个 **大于等于** `x` 的位置，用 `x` 替换它。
   - *为什么要替换？* 因为更小的结尾意味着后面能接上更大的数的可能性更大（贪心思想）。

**例子演示 (`nums = [2, 5, 3, 7]`)：**
1. `x=2`: `tails = [2]`
2. `x=5`: `5 > 2`, append -> `tails = [2, 5]`
3. `x=3`: `3` 替换 `5` -> `tails = [2, 3]` (虽然长度没变，但结尾变小了，更有潜力)
4. `x=7`: `7 > 3`, append -> `tails = [2, 3, 7]`
5. 结果：长度为 3。
